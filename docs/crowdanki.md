# CrowdAnki compatibility

AnkiIO was informed by CrowdAnki's public MIT-licensed behavior: nested `children`, root `note_models`, `deck_configurations`, `crowdanki_uuid`, note `guid`, ordered `fields`, tags, and sibling media filenames. The implementation is independent and is not a port.

Supported: deck hierarchy, note models, ordered fields, editor direction/stickiness/font settings, templates, browser-specific formats, CSS, cloze kind, notes, tags, GUIDs, and media references. Not supported: CrowdAnki Git workflows, personal fields, interactive model remapping, arbitrary Anki dictionary keys, scheduling, review history, or claims of byte-for-byte compatibility. CrowdAnki attribution is retained in the repository's `THIRD_PARTY_NOTICES.md`.
